- 中国城市 debug，比如 Hong Kong、Shanghai、Urumqi 等
- 深度学习训练与预测文件
- Tavily 搜索、LLM 调用、日志、输入数据、输出结果
- 很多中间产物和调试文件

- [city_scoring.py](</d:/Work/Resilience LLM_ZYA/LLM_form_Dr.Zhang/city_scoring.py>)

`urban-resilience-parliament-hk-case-main`
是后面 GitHub 给你的整理版，目标更明确：**展示 Urban Resilience Parliament 这个方法在香港案例上的可复现流程**。它有标准 GitHub 项目结构：

- `requirements.txt`

它重点不是完整大规模数据 pipeline，而是一个**香港 case study demo**：8 个 LLM，两轮评分，中间有 Consul audit，然后输出香港 18 个 resilience indicators 的结果。

| 文件夹 | 性质 | 作用 |

| `LLM_form_Dr.Zhang` | 早期/内部实验工作目录 | 做城市因子评分、AAL/Exposure 降尺度、深度学习、调试中国城市等 |

它们确实有继承关系：比如两个目录都有 `llm_client.py`、`tavily_websearch.py`，说明新项目复用了旧项目里的 LLM 调用和搜索基础。但新项目不是旧项目的完整升级版，而是**从更大的研究代码中抽取出来的一个方法展示版/论文 demo 版**。

一句话总结：

这三者的关系可以这样理解：

**1. `SCS_paper_v28_江宇杰二作_20260528/main.tex` 是论文正文**
它是要投/写给 *Sustainable Cities and Society* 的文章，题目是：

> *A Multi-LLM Agent Parliamentary Architecture for Constructing the City Infrastructure Resilience Index*

这篇文章讲的是完整研究方法：**MPA，多 LLM 智能体议会架构**，用于构建城市基础设施韧性指数。论文里说应用对象是 **50 个全球代表性城市**，不是只有香港。

**2. `urban-resilience-parliament-hk-case-main` 是这篇论文公开释放的代码/demo 仓库**
论文的 Data/Code Availability 部分明确写了：

> The MPA framework and a fully reproducible Hong Kong case study are released ... at GitHub

也就是说，这个 GitHub 仓库是论文配套的**公开可复现案例**。但它只放了香港 case，不放完整 50 城市数据，所以 README 才说 `no large-scale dataset.`

它和论文的关系是：

> 论文方法的公开演示版 / reproducible demo / supplementary code repository。

**3. `LLM_form_Dr.Zhang` 更像是论文形成过程中的早期/内部实验材料**
它里面有城市评分、GIRI 因子、AAL/Exposure 降尺度、深度学习、debug logs 等，范围比香港 demo 更杂。它可能是这篇论文或相关后续论文 pipeline 的早期开发包、实验包、素材包。

但它不像 `urban-resilience-parliament-hk-case-main` 那样是论文中明确声明的公开仓库，也不是一个整理好的 submission artifact。

它能实现的事情大概是：

**1. 给城市生成基础画像和评分**

旧版本里有 `city_scoring.py`、`main_china_18_factors_debug.py`、`tavily_websearch.py`、`llm_client.py`。这套东西可以让 LLM 对某个城市做信息搜集和评分。

- 输出城市 GDP、人口等基础统计；
- 给城市打若干个 0-1 分数，例如人口规模、经济规模、教育科研、交通枢纽、公共设施、灾害暴露等；
- 保存 LLM 原始回答和整理后的结果。

所以这部分可以做成一个项目：

> **基于 LLM 的城市基础韧性因子自动评估系统。**

**2. 把国家级 18 个 GIRI 因子推到城市级**

旧版本里有这份说明：

> 城市级因子不是从零编造，而是以国家级 GIRI 因子为 baseline；只有当城市有明确证据时，才允许在有限范围内修正。

- 城市证据充足时，最多能修正多少？
- 缺少证据时，如何退回国家 baseline？

所以这部分可以做成：

> **国家级 GIRI 韧性指标到城市级指标的 LLM-assisted downscaling 工具。**

```text
生成城市基础因子：人口、GDP、交通、教育、公共设施、灾害暴露等
        ↓
把国家级 18 个韧性因子修正为城市级 18 个因子
        ↓
把国家级 Exposure 和 AAL 按城市权重分解到城市
        ↓
输出城市级 resilience indicators、exposure、AAL、debug logs、预测结果
```

一句话总结：

**旧版本真正能做的事情，是把“国家级灾害韧性/损失数据”借助 LLM 和规则模型推到“城市级”，从而生成城市层面的韧性评分、暴露分解和损失估计。它更像一个内部研究 pipeline，而不是一个已经包装好的公开 demo。**
